Using the BroadMask Extension

Installation

The latest Chrome and Firefox extension for Mac OSX (Universal, tested on 10.7+) and Linux 32/64-bit versions is available at:

https://github.com/oliverguenther/broadmask/downloads

Download the appropriate extension file and drop the extension file on your browser. For Chrome, drag the file onto the Chrome's extensions page (Tools > Extensions).

Dependencies

For file encryption and key distribution, BroadMask depends on an active GnuPG installation.

Using the Extension

Profiles

Prior to using BroadMask for profile management, you need to create a local profile, which is essentially a local storage for all generated keys.

You can create a Profile in the Settings tab. Enter a name and select one of your local PGP keys, which will be used to encrypt your data. Once created, click the Use this Profile link to activate the profile (Will open GPG Pinentry program unless you cached the credentials.)

Social Networks

After setting up a profile, you'll need to authorize Facebook to access the API on your behalf. You can do so on the Social Networks tab.

Note: While BroadMask does not require basic permissions (does not read/access any personal information), BroadMask depends on two extended permissions:

  • publish stream Reason: Publish encrypted posts to your wall
  • read stream: Filtering your wall for incoming BroadMask posts

If you wish to privately share media files, you additionally need a Google account and authorize it to access Google PicasaWeb for untrusted storage. Any media files you share are converted to a data URL and encrypted before uploading it to Picasa.

Manage Groups

Now you're ready to create your first private groups by entering a group name (which is not shared, but only used for easier identification). Select one of the following instance types:

  • Broadcast Encryption: A broadcast encryption system uses pairing-based cryptography to derive a symmetric key. Use this variant if you wish to encrypt content for a varying subset of your friends

  • Shared-Key Encryption: Share the content with a fixed symmetric key. Use this scheme if you always share with the same number of receivers.

Then you can add users to your group by entering names of your Facebook friends, whose names have been cached by the extension. If you're ready, click the Create instance button, which sets up the instance and generates the necessary keys.

Share Content

Once you've created one or more private group instances, you may enter private messages (along with media files) in the Share Content tab. Simply drag & drop files on the text message field or select media files. Select the group instance to share with, and hit the upload button.
The progress of media file upload is displayed and the post is shared on your Facebook wall using the Facebook SDK's default sharing setting (which currently defaults to all friends).

Note: Sharing media files requires you to authorize your Google account beforehand. Picasa's file upload limit is 20 MB.

Viewing Content

The Extension will parse BroadMask wall posts directly on Facebook whenever you visit the Facebook pages. If the necessary instance details are found, it will download the post (and optionally, and media linked within), decrypt it and inject it back in the wall post.

Key distribution

In order for your friends to decrypt your posts, they need the BroadMask extension and your private group instance details (a 32-byte key for Shared-Key encryption, and Public/Private keypair for Broadcast Encryption).

In the Manage Groups tab, clicking on an instance will display all members in that particular group. You can manually share these details to your friends by copying it and forward to them using a secure channel (e.g., encrypted email).

But obviously, this becomes cumbersome for a large number of friends. Thus, you may use OpenPGP's infrastructure for exchanging keys directly within Facebook! Open the Key Mapping tab. At the bottom of this tab, you may upload a fingerprint of your PGP public key as a Facebook wall post. The extension will also fetch PGP public keys of your friends they shared in the same manner. Using the Update manually button, you can force a refresh PGP fingerprints other people shared on their Facebook wall.

Note: If you already possess the PGP key of some Facebook friend of yours, copy the PGP fingerprint and use the Create Mapping form to import the mapping to your profile.

If a PGP fingerprint is found, the associated key is imported to your GnuPG keyring. Important: Validate the correctness the fingerprint of your friend's PGP key! If you don't trust it, remove it from the mappings list.

For all friends with a known PGP fingerprint, you can share the instance details (from the Manage Groups tab), encrypted using the matching PGP public key, as a Facebook wall post. Once the friend visists Facebook and sees your post, the extension grabs the details, decrypts it and adds the instance---allowing her to decrypt your post!
